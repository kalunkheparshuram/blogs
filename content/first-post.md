## Hello, World!

Welcome to my first blog post.

I created this space to document what I learn, build, break, test, and
discover along the way.

My primary interests are **cybersecurity, penetration testing, bug
bounty hunting, red teaming, networking, Linux, Windows, Active
Directory, and programming**.

Instead of keeping everything buried inside personal notes, I want to
use this blog as a public knowledge base where I can document practical
experiments, technical concepts, research, and lessons learned.

## What I'll Write About

You can expect posts covering topics such as:

-   Cybersecurity and penetration testing
-   Web application security
-   Bug bounty hunting
-   Red teaming
-   Active Directory security
-   Linux and Windows
-   Networking
-   Burp Suite and web security tools
-   Vulnerability research
-   CTF and lab walkthroughs
-   Programming and automation
-   Security news and interesting vulnerabilities
-   Notes from projects and experiments

Some posts may be detailed technical write-ups, while others may simply
be short notes about something useful I learned.

## Why I'm Writing This Blog

Cybersecurity involves a huge amount of continuous learning.

Reading documentation and watching courses can help, but I believe real
understanding comes from actually **building labs, testing things,
making mistakes, troubleshooting problems, and documenting what
happened**.

Writing also forces me to explain concepts clearly instead of assuming
that I understand them.

This blog is therefore both a personal learning archive and a place
where I can share useful information with others.

## Built with GitHub

The blog itself is intentionally simple.

Posts are written as Markdown (`.md`) files and stored in my GitHub
repository. The website retrieves these files and automatically displays
them as blog posts.

That means publishing something new can be as simple as:

1.  Write a Markdown file.
2.  Add it to the repository.
3.  Commit and push it to GitHub.
4.  The website loads the new post.

No complicated CMS required.
